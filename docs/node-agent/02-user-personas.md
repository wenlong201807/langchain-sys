# 用户画像与使用场景

## 核心用户画像

### Persona 1：独立内容创作者 - 小鱼

| 属性 | 描述 |
|------|------|
| 年龄 | 28 岁 |
| 职业 | 自媒体博主 / 自由撰稿人 |
| 日常工具 | 微信公众号、小红书、飞书文档、Notion |
| 痛点 | 每天需要产出大量内容，选题调研耗时；写作风格需要保持一致；多平台发布格式不同 |
| 期望 | 一个能帮她调研热点、整理素材、按她的风格生成初稿、适配多平台格式的 AI 助手 |
| 付费意愿 | ¥39/月可接受，如果能节省每天 2 小时 |

**核心场景：**

```
场景 1：热点选题调研
小鱼：「帮我搜一下最近一周科技圈的热点话题，整理出 5 个适合做深度解读的选题」

场景 2：风格化内容生成
小鱼：「按照我之前的写作风格，把这个选题写成一篇小红书笔记」
Agent：[读取长期记忆中的写作风格偏好] → [生成符合小红书格式的内容] → 流式输出

场景 3：多平台适配
小鱼：「把这篇笔记改成公众号文章版本和微博短文版本」
Agent：[理解原文核心观点] → [按不同平台特点改写] → 输出多版本内容
```

## 用户旅程地图 (MVP 核心流程)

```
注册/登录
  │
  ├─→ 首次引导
  │     ├─ 设置个人偏好（职业、常用场景、语言风格）
  │     ├─ 创建第一个知识库（可选）
  │     └─ 体验示例对话
  │
  ├─→ 日常使用
  │     ├─ 智能对话（核心）
  │     │    ├─ 文本对话 → Agent 推理 → 工具调用 → 流式回复
  │     │    ├─ 上传文件 → 解析 → 基于内容问答
  │     │    └─ 语音输入 → 转文字 → 对话（移动端）
  │     │
  │     ├─ 知识库管理
  │     │    ├─ 上传文档（PDF/Word/Markdown/TXT）
  │     │    ├─ 导入网页 URL
  │     │    └─ 手动添加 Q&A 对
  │     │
  │     └─ 历史记录
  │          ├─ 按时间线浏览对话
  │          ├─ 搜索历史对话
  │          └─ 收藏/导出重要回复
  │
  └─→ 进阶功能
        ├─ 自定义 Agent 角色
        ├─ 团队协作（Team 版）
        └─ API 调用（Enterprise 版）
```

## 用户故事 (User Stories)

### P0 - MVP 必须

| # | 用户故事 | 验收标准 |
|---|---------|---------|
| US-01 | 作为用户，我想通过文字与 AI 助手对话，以便获得问题的回答 | 支持多轮对话；流式输出；响应时间 < 3s |
| US-02 | 作为用户，我想让 AI 帮我搜索互联网信息，以便获取最新资讯 | Agent 能自主决定何时搜索；返回结果带来源链接 |
| US-03 | 作为用户，我想上传文档建立知识库，以便 AI 基于我的资料回答 | 支持 PDF/Word/Markdown/TXT；上传后 < 60s 可用 |
| US-04 | 作为用户，我想让 AI 记住我的偏好和上下文，以便下次对话延续 | 跨会话记忆用户偏好；同一会话保持上下文 |
| US-05 | 作为用户，我想在手机上也能使用，以便随时随地获取帮助 | 微信小程序可用；核心功能与 Web 端一致 |
| US-06 | 作为用户，我想查看和管理历史对话，以便回顾和导出内容 | 对话列表；搜索；删除；导出为 Markdown |

### P1 - MVP 优先

| # | 用户故事 | 验收标准 |
|---|---------|---------|
| US-07 | 作为用户，我想自定义 AI 的角色和风格，以便适配我的使用场景 | 可设置系统提示词、名称、头像 |
| US-08 | 作为用户，我想让 AI 生成结构化内容（表格、列表），以便直接使用 | 支持 Markdown 表格、列表、代码块渲染 |
| US-09 | 作为用户，我想在对话中上传图片让 AI 分析，以便处理视觉信息 | 支持图片上传；多模态模型识别图片内容 |
| US-10 | 作为用户，我想对 AI 的回复进行反馈（点赞/点踩），以便 AI 改进 | 每条回复有反馈按钮；反馈数据入库 |

### P2 - 后续迭代

| # | 用户故事 | 验收标准 |
|---|---------|---------|
| US-11 | 作为团队管理员，我想创建团队共享知识库 | 团队空间；权限管理；共享 Agent |
| US-12 | 作为用户，我想通过语音与 AI 对话 | 语音输入转文字；TTS 语音回复 |
| US-13 | 作为开发者，我想通过 API 调用 Agent 能力 | RESTful API；API Key 管理；调用量统计 |
