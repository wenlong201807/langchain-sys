# MVP 开发路线图与排期

## 总体计划

- **开发团队：** 1 名全栈工程师（精通 Node.js / React / UniApp）
- **总工期：** 10 周（2.5 个月）
- **开发方式：** 敏捷迭代，每周交付可演示版本
- **里程碑：** 5 个 Sprint，每个 Sprint 2 周

---

## Sprint 总览

```
Week 1-2   │ Sprint 1: 基础设施 + 核心对话
Week 3-4   │ Sprint 2: 知识库 + 记忆系统
Week 5-6   │ Sprint 3: 移动端 + 工具增强
Week 7-8   │ Sprint 4: 用户体系 + 上线准备
Week 9-10  │ Sprint 5: 后台管理系统（RBAC + Admin SPA）
───────────┼───────────────────────────────
Week 11    │ 内测 + Bug 修复
Week 12    │ 公测上线 🚀
```

## 技术风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| OpenAI API 不稳定 | 中 | 高 | 多模型降级策略（GPT-4o → Claude → Gemini） |
| 小程序 SSE 兼容性 | 高 | 中 | 提前验证 RequestTask 方案，准备轮询降级 |
| 向量检索性能 | 低 | 中 | pgvector HNSW 索引，MVP 数据量可控 |
| 微信小程序审核 | 中 | 高 | 提前准备资质材料，预留 1 周审核时间 |
| 用户数据安全 | 低 | 极高 | Prisma ORM 防注入，HTTPS 传输加密，定期备份 |
| 单人开发瓶颈 | 中 | 中 | 优先核心功能，非核心功能推迟；充分利用 AI 编程工具提效 |
| RBAC 权限漏洞 | 低 | 极高 | 后端强制校验 + 前端隐藏双重保障；审计日志全量记录 |
| 管理后台越权访问 | 低 | 极高 | 独立域名 + IP 白名单/VPN 限制 + 短 Token 有效期 |
